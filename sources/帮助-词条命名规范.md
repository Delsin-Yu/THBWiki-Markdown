## 基础规范
- 一般情况下，所有词条都应当以 **简体中文** 命名，仅在没有公认的中文名时方可使用其他语言文字。作品、机构名称或人名可以根据使用习惯保留原名或假名对应的英文名。

: 例如：[上海爱丽丝幻乐团](./上海爱丽丝幻乐团.md)（简体译名）、[SOUND VOLTEX](./SOUND_VOLTEX.md)（保留英文名）

- 词条名称应当对内容有一定的概括性，而非意义不明的文字。尤其是上传文件时文件名称必须包含对文件内容或用途的正确描述。

: 例如：[其他ZUN参与的作品](./其他ZUN参与的作品.md)（直接描述）、[东方文果真报/读物/M因M而死](./东方文果真报-读物-M因M而死.md)（按层次概括）、[文件:博丽灵梦（天空璋立绘）.png](./文件-博丽灵梦（天空璋立绘）.png.md)（文件出处及用途）、[文件:妖妖梦森罗结界爆炸.jpg](./文件-妖妖梦森罗结界爆炸.jpg.md)（文件内容）

- 除非原文包含或进行必要的区分，不应在条目名称添加修饰词。
- 命名应当尽量减少标点符号的使用，不要添加原文没有的书名号（《》）或引号（「」），并使之符合标点符号使用规范 (未找到链接)。

: 例如：[式神「八云蓝」](./式神「八云蓝」.md)（引号是原文的一部分，亦需要和[八云蓝](./八云蓝.md)区分，所以可以保留）

- 对于全角空格，Wiki系统会自动转为半角空格；而连续空格都会自动消减到只剩一个。
  - 对于这种情况可以使用displaytitle函数 (未找到链接)来修正显示。

- 词条名称应当避免过长（100字节以上），Wiki系统下词条名极限长度是255字节，相等于85个中文字（全形符号）。
- 基础规范是所有词条命名均需要遵守的规范（有特殊说明的除外），后文的详细规范附加在基础规范之上。

  
以下词条命名 **不应出现在THBWiki中** ：
  

```
萎えラジ
千儿丿
巫女博丽灵梦
《东方香霖堂》
```

  
应当使用：
  

```
[枯萎radio](./枯萎广播.md)
[琪露诺](./琪露诺.md)
[博丽灵梦](./博丽灵梦.md)
[东方香霖堂](./东方香霖堂.md)
```

  
如有确切需要使用这种别名可以使用[重定向](./帮助-特殊代码.md)功能。
  


## 内容页面
  
消歧义页面及重定向页面无须遵守以下内容。
  

- 应当将公认的名称／译名作为内容页面，并将其他常用称呼使用[重定向](./帮助-特殊代码.md)功能重定向到相应的主词条上。
- 对于拥有子页面的页面，应当尽量减少词条名称长度，将核心短语作为内容页面，将拥有英文（或日文／中文）后缀（前缀）的全称设为重定向页面。
- 不应使用缩写作为内容页面名称。
- 使用“XXXX:YYYY”格式命名的页面，应保证XXXX为已有名字空间之一。
- 对于目前不需要消歧义（没有重名）的页面，不应在名称后添加“（XX）”。  
如“donjuan（视频）”应为“[donjuan](./donjuan.md)”。
- 当页面内容过多（超过300,000字节）时，应当考虑将页面分为多个子页面。


### 子页面
- 当父页面“XXXX”不存在时，不应创建“XXXX/YYYY”子页面。
- 子页面仅应作为父页面的补充而存在，而并非凡是有从属关系的页面都设为子页面。  
如“[秋霜玉](./秋霜玉.md)”页面，显然不应设为“西方project/秋霜玉”。
- 对于有多个父页面的页面，应当进行讨论并以讨论结果确定命名。  
如“[游戏攻略/东方红魔乡](./游戏攻略-东方红魔乡.md)”或“东方红魔乡/攻略 (未找到链接)”页面。
- 歌词命名空间不能创建子页面，所以歌词相关的其他内容也应该写在歌词页面上。


### 官方作品相关
- 各类词条在创建时，大多已有THBWiki的确定译法，请根据[官方角色列表](./官方角色列表.md)等页面设置页面名称。
- 对于部分已有习惯译名却不符合规范的名称，应当进行讨论并以讨论结果确定命名。  
如“[东方文花帖DS](./东方文花帖DS.md)”（已确定）。
- 出现重名时，游戏词条保留原名，其余词条在名称后添加（）标注。  
如“[东方文花帖](./东方文花帖.md)”与“[东方文花帖（书籍）](./东方文花帖（书籍）.md)”。
- 附带文档词条，格式为“附带文档:游戏名/文档名”，文档名应参考[模板:附带文档信息](./模板-附带文档信息.md)。
- 游戏对话词条，格式为“游戏对话:游戏名/自机类型”，对于不同的机体的词条命名，以讨论结果为准。
- 符卡词条，建立「」内的文字为名称的页面，并建立包含「」的全名重定向页；仿照[元netawiki](https://seesaawiki.jp/toho-motoneta_2nd/)，对于近似并有对应关系的符卡进行合并，根据习惯称呼对合并页面命名。

  
待讨论的问题：
  

- 名称相同但显然应区分为多个页面的页面（如：东方三月精ESOV）要怎么才能既区分又不使名称过长？
- 非英文真的要放到名称上吗？如：[Guarneri del Gesù](./Guarneri_del_Gesù.md)
  - 既然CJK文字等非ASCII码文字都已作为词条名称，这一点应有必要。



### 二次同人作品
- 由SMW抓取列表方式要求，二次作品应当将 **原文的全称** 作为主条目，并将简称／中文译名作为重定向页面。
- 二次作品页面不应当使用“制作方/名称”或“大作品/小作品”的格式。（除非是同人周边。）
- 专辑形式发布的二次同人音乐不得为每首曲目创建词条，只允许为专辑建立词条，以专辑名命名。
- 在标题中出现的半角斜杠/，应当替换为全角斜杠／如：（IA-STYLE／Toho-J／Fantasmix）
- 对于歌词页面同名碰撞，同名碰撞会在“社团歌词进度”里有多于一个的“编辑专辑”链接，remix较少的团的多链接／常用词汇的标题要特别留意是否含同名曲。以 歌词:曲名（社团）创建页面，并在所有社团的首发曲目后都加上|歌词 = 曲名（社团）
- 由于同人周边容易重名，所以词条名称应采用子页面，以“社团名/系列名/作品名”（注意：这里的/是半角斜杠，而非全角斜杠／。）的格式命名，作品名可以只写相应角色名。如果没有系列，可以直接使用“社团名/作品名”的格式。
- 二次同人作品的附带文档和游戏对话词条请勿使用“附带文档:游戏名/文档名”和“游戏对话:游戏名/自机类型”的格式。


## 重定向／消歧义页面
- 建立一个页面时，应当为它的原文／汉语名称／缩写／全称设置重定向。
- 如果一个页面名称拥有难以打出的符号，或易打错的符号，可以为其设置近似拼写的重定向。
- 由于Wiki对全半角、繁简日本汉字、大小写不敏感，因此不必设置半角跳全角、小写跳大写等的重定向。
- 不应当设置目标页面不存在的重定向页面。
- 不应当设置一个既非原文／汉语名称／缩写／全称／近似拼写，又没有链入页面的重定向。

- 何种程度应当建立重定向页面 而非建立单独页面？（Flower～战车）


## 模板
- 模板名称应简明扼要地说明模板用途，并符合惯有命名格式和用词，方便记忆。


## 分类
- 不应当设置以“分类:”开头的重定向页面，因为根本不会有任何效用。
- 添加及创建分类时必须确定该分类不是两个更独立具体的分类的交集。

: - 比如“2016年同人专辑”是“2016年”和“同人专辑”的交集，所以应使用“2016年”和“同人专辑”，而不是“2016年同人专辑”。
- 但也有例外的情况，比如“同人专辑”是“同人”和“专辑”的交集，但“专辑”内包含的只有并只能是“同人专辑”，所以会使用“同人专辑”，而不是“专辑”和“同人”。


- 同人社团名必须跟主词条名一致（包括括号后缀），繁简也必须一致。
- 同人游戏并不应当单独设置同名分类。


## 文件
- 任何作品的封面图片文件名必须是“文件:作品名封面.文件扩展名”的格式，单行本、单话或者特别版封面均需要在“封面”二字前注明，比如“文件:东方红魔乡封面.jpg”和“文件:东方铃奈庵4卷封面.jpg”
- 不要添加原文没有的书名号（《》）或引号（「」）。
- 应当保证新文件名与已有的同类文件命名格式相同。


## 游戏对话和附带文档
- 仅东方官方游戏、西方、连缘等可以使用游戏对话和附带文档的命名空间，即其游戏对话和附带文档词条可以使用“游戏对话:”“附带文档:”开头。
  - 原因是使用这些前缀会导致与主词条失去关联。

- 东方官方游戏务必使用。
