## 收录标准
  
以下类型的地点应被收录：
  

- 在官方游戏（包括结局）出现过；
- 在官方设定集出现过；
- 在漫画用方形气泡专门标出；
- 在秘封CD中拥有专门描述该地点设定的文本。

  
其中，只有较为独立的子地点应被收录，虽然出现位置符合标准但是独立性过弱的场景地点则不被收录。
  

- 例如，博丽神社境内的相关内容应在博丽神社词条内直接介绍，但三月精的家（新居）则作为子地点被另行收录。

  
收录的地点中，有官方游戏或官方设定集的图像的应在独立页面中介绍，没有的应附在上一级地点的附属场景中介绍。  

包括未收录的零碎地点在内，所有地点都应在[官方场景](./官方场景.md)内简要说明，并配上一张整体外观图（若只有细节图则配细节图）。对于被收录的地点，最后加上“&lt;big&gt;详见[[该地点]]。&lt;/big&gt;”。
  

## 基本信息填写
### 名称
  
 **名称** 直接填写中文名。一般情况下要与页面标题一致。  

 **日文名** 至少需要有原作中出现过的准确的地点名称。如果可以的话，还请在括号中标注上假名拼法和其读音。  

 **英文名** 不是必须的，但是如果可以的话最好还是添加上。大部分地名的英文名都可以在[TouhouWiki](https://en.touhouwiki.net/wiki/Touhou_Wiki)中找到。  

若有其他日文名或不同的译名，使用属性 **其他日文名** 与 **其他译名** 。
  

### 类别
  
需要满足以下名称规则：
  

- 若为人造的建筑物，则类别统一为 **建筑** 。可以在后面使用括号备注上建筑的类型。如：  
博丽神社：建筑（神社）
- 若为生物的聚居地，则类别统一为 **聚居地** 。可以在后面使用括号备注上聚居地的类型。如：  
人类村落：聚居地（村落）
- 若为连接两地点间的通道，则类别统一为 **道路** 或 **通路** 。非传统意义上的道路的都归作通路，可以备注出通路的类型。如：  
妖怪兽道：道路  
幻想风穴：通路（洞穴）
- 若为江河、湖泊、溪流等水文景观，则类别统一为 **水体** 并备注上所属的类型（河流/溪流/湖泊/瀑布/运河等）。如：  
雾之湖：水体（湖泊）
- 若为森林、草原等植物景观，则类别统一为 **植被** 并备注上所属的类型（森林/草原等，包括人工植被）。如：  
魔法森林：植被（森林）
- 若为幻想乡外的其它世界，则类别统一为 **另一个世界** 。
- 无法归类为以上任何一类的，请自行组织语言进行描述。

  
一个地点可以同时拥有多个类别。如：
  

<unsupported html=blockquote>
  
难以归类的地点（如核聚变炉）可以选择不填写该项。
  

### 出场信息
- 有两个属性： **出现作品** 与 **出现作品说明** 。每个属性的各个子项之间用逗号分隔，两个属性的相同位置子项一一对应。例如，输入以下内容：

```
| 出现作品 = 东方求闻史纪，东方红魔乡
| 出现作品说明 = ，二面
```

 **出现作品** 需要写上所有有该地点出现的官方作品。按照整数作-小数点STG-黄昏作STG-漫画/小说-设定集-音乐CD的顺序排列。  

 **出现作品说明**   

对于整数作（对战作除外），需要写上具体出现的面数。如果道中与关底的场景有区别需要加以体现。如
  

- 畜生界：[东方鬼形兽](./东方鬼形兽.md) 五面道中

  
对于小数点STG作，按照各作特有的表示方法（如Level X/第X日/星期X等）标明。特别地，对于大战争，直接用正篇/EX面表示。如
  

- 幻想乡上空：[弹幕天邪鬼](./弹幕天邪鬼.md) 第五日  
[秘封噩梦日记](./秘封噩梦日记.md) 噩梦周四

  
对于对战stg以及格斗作，不需要标明出现位置。  

对于漫画或小说作品，原则上应标上出现的话数。  

设定集作品不用标明出现位置。  

音乐CD类作品需要标明出自哪一个音轨的附带故事。
  

### 所在地
  
如果设定中有详细的所在位置则填入，否则留空。
  

### 相关角色
  
只要是有关的角色都可以填入，并用括号备注出详细关系。
  

## 相关设定
  
主要介绍居住者情况、经常举行的活动以及其他在原作文本中提及的设定，并用可读性高的语言整理。  

请务必遵守客观、公正、准确的原则。如不加任何说明，应保证内容均属于 **官方作品信息的无推断的整理** ，避免编写无根据的个人看法。

## 一设资料
  
用次一级的标题将出自不同官作的设定分开，标题的命名使用作品的全称。  

官作的排布顺序与出现作品的排布顺序相同。  

需要将设定的内容进行裁剪，将地点描述关系不大的部分删去，用 **……** 代替。  

将一设资料中出现的内容进行整理，放到 **相关设定** 中。  

将出自官方作品的图片中包含的信息整理后，放到 **结构与造型** 的设定点下。
