## 基础规范
- 本维基是东方Project主题维基，并非收录广泛的Wikipedia，词条内容必须符合[THBWiki收录方针](./THBWiki-收录方针.md)，如果想编写不符合方针的词条请至[Wikipedia](http://zh.wikipedia.org/)。
- 在本维基编写词条必须本着客观的原则，要尽可能保证词条内容准确性，模棱两可或无法判断准确性的内容请使用[出处标记](./帮助-特殊代码.md)。
- 请不要将自己的主观看法与客观内容混杂在一起。
- 转载必须标明出处，有参考资料的请使用[参考资料代码](./帮助-常用代码.md)标注参考资料。
- 基础规范是所有词条内容均需要遵守的规范（有特殊说明的、得到管理员同意的除外），后文的详细规范附加在基础规范之上。

## 标点符号使用规范
- 本Wiki正文以中文为主，逗号（，）和顿号（、）以中文的用法为准。
- 中文内容的括号请使用全形括号（），英文内容的括号请使用半形括号()。
- 省略号无论在中文日文或西文都最好使用…，……（连续两个）也行，用中文输入法按Shift+6就能打出来，同时请尽量避免使用3到6英文句点（......）。
- 乘号、叉号、以及表达合作和CrossOver的符号均为×，一般中文输入法输入cha都会出现，请勿使用x和X（大小写拉丁字母x）。
- 使用空格时，若半形空格不足以满足需求请使用全形空格或&amp;nbsp;，直接输入多个半形空格是无效的。

### 引号
- 引号常用于标示引用、着重、特别用意，可详见中文维基百科：[引号](https://en.wikipedia.org/wiki/zh:引号)。
- 引号的使用在不同地区及不同场合中的使用差异较多。
- 以下列出应该使用（和不应使用）的引号（和假引号）：

: - “”， **逗双引号** 

: : 在多数简体中文环境会自动调整为全形大小而在西文环境会自动调整为半形大小，可用于中文、西文及日文原文如此的情况。


- 「」， **直角单引号** 

: : 和逗双引号一样可以用于中文内容中，在逗引号显示为半形大小的环境中建议使用，在中文纵排的情况下应当使用。


- 『』， **直角双引号** 

: : 除日文原文如此或多重引号分级的情况外， **请尽量避免使用** ，不建议作为第一级引号。


- ‘’， **逗单引号** 

: : 除原文如此的情况或多重引号分级外， **请尽量避免使用** ，不应作为第一级引号。


- ’， **逗撇号** 

: : 同逗单引号，除引用原文如此、西文书写的情况外， **请尽量避免使用** 。


- "， **直引号** 

: : 除引用原文如此、西文书写的情况外， **请尽量避免使用** 。


- '， **直撇号** 

: : 单独出现时除原文如此、西文书写的情况外， **请尽量避免使用** ；绝不应该将其连续作为引号使用。


- `， **重音符** ，又称为 **反单引号** 

: : 在电脑程序以外的地方都不应该使用，至少应替换成 **逗单引号** 。


- 其他特殊引号如«»、„“等，除非引用原文如此， **应尽量避免使用** 。


### 间隔号
- 间隔号一般用于分隔非汉语人名，可详见中文维基百科：[间隔号](https://en.wikipedia.org/wiki/zh:间隔号)，另外还有中文维基百科：[Help:如何输入间隔号](https://en.wikipedia.org/wiki/zh:Help:如何输入间隔号)的说明。
- UTF8下有大量间隔号和与间隔号相似的符号，一般对输入法不熟悉、对这种标准没啥强迫性的人根本区分不出来。然而对于电脑来说那些符号都是不一样的，无法互相搜索。
- 大部分字体和文字寻找工具都会“好心”地把其中一部分的间隔号和假间隔号混为一谈，以致在需要纠正时无从区分。
- 以下列出应该使用（和不应使用）的间隔号（和假间隔号）：

: - ·， **间隔号** 

: : 这个是 **正确的间隔号** ，该符号在大部分简中字体中会自动调整为全形大小，中文内容必须使用此 **间隔号** 。


- ．， **全形西文句号** 

: : 很多时候会被日本人当作句号和省略号使用，只允许存在于日文原文内容中或是中文排版中用以对应西文半形句号的情况。


- ・， **全角假名中点** 

: : 纯粹源自JIS编码，在日文中和 **间隔号** 有着同等用法，只允许存在于日文内容中，在其他语言中应替换成正确的 **间隔号** 。


- •， **项目符号** 

: : 表示一个项目的开始，不应该当作 **间隔号** 使用，在Wiki中理应使用*和li代替。


- ‧， **连字点** 

: : 在任何地方都不应该使用，应替换成 **间隔号** 。


- ∙， **数学符号** 

: : 在数学公式以外的地方都不应该使用，应替换成 **间隔号** 。


- ⋅， **数学符号** 

: : 在数学公式以外的地方都不应该使用，应替换成 **间隔号** 。


- ･， **半角假名中点** 

: : 在任何地方都不应该使用，至少应替换成 **全角假名中点** 。



### 直线连接号
- 直线连接号是用于表示连接、起止、流程的符号，可详见中文维基百科：[连接号](https://en.wikipedia.org/wiki/zh:连接号)。
- 直线连接号也有和间隔号同样的问题，以及存在着许多和直线连接号相似的「直线」，在使用时需要仔细区分。
- 以下列出应该使用（和不应使用）的直线连接号（和非直线连接号）：

: - —， **（单）破折号（Em dash）** 

: : 用于中文、西文表示语气转折等；在日文中也可使用，但由于大多数日文字体显示问题更推荐使用水平线。


- ——， **（双）破折号** 

: : 两个单破折号并排，在简中输入法中按“Shift＋-”就能打出来，同样可以用于中文表示语气转折等。


- ―， **水平线** 

: : 在大多数日语输入法中显示为「ダッシュ」的符号实际就是这个，用于日文表示语气转折等，但在其他语言中不应该使用。


- -， **连字暨减号** 

: : 可用于西文中作为连字号、减号等场合。


- –， **连接号（En dash）** 

: : 用于西文中表示范围连续。


- −， **减号** 

: : 当用减号，不应用于其他用途。


- ー， **日文长音** 

: : 用于日文中表示长音，不应用于其他用途。


- ‐， **连字号** 

: : 标准连字号，不应用于其他用途。


- 一， **汉字一** 

: : 非标点符号。



### 波浪连接号
- 波浪连接号在东方系列中经常作为标题和副标题的分隔符使用，其他用法可详见中文维基百科：[连接号](https://en.wikipedia.org/wiki/zh:连接号)。
- 波浪连接号也有和间隔号同样的问题，只是没间隔号那么严重。
- 以下列出应该使用（和不应使用）的波浪连接号：

: - ～， **全形波浪连接号** 

: : 一般中文输入法输入bolang都会出现，东方官方所有命名使用的都是这个，任何语言均可使用。


- ~， **半形波浪连接号** 

: : 英语中比较常用，同样任何语言均可使用。


- 〜， **浪纹号** 

: : 在多数字体中显示为倒转的 **全形波浪连接号** ，纯粹源自JIS编码，在任何地方都不应该使用，应替换成 **全形波浪连接号** 。

- 常见类别的词条建议在顶部放置[词条头标](./帮助-特殊代码.md)。
- 有歧义的词条名建议在词条顶部做[消歧义](./帮助-特殊代码.md)处理。
- 在编写前请先阅读帮助中的[常用代码帮助](./帮助-常用代码.md)与[高级代码帮助](./帮助-特殊代码.md)以使用更多的功能。
- 本站记录二次同人作品主要是作为索引，收录的是作品信息，给出作品的通贩、下载、观看渠道等。  
所收录作品务必是能确定已经发布或者即将发布的，客观存在的，能让大众知晓的作品。  
如果尚处于尚未成形企划阶段，请勿收录（作品是实实在在的，企划是虚的，谁知道会不会 ~~咕咕~~ ）。
  - 本站的定位是资料站，对信息进行汇总，不是宣传分发平台，不是社交媒体平台，也不是作品创作和发布平台。 ~~（所以写了也不一定有人看）~~ 

- 本站收录二次同人作品的信息，但不收录作品本身，不需要将作品原文件上传。
  - 例如，某个同人志DL版的PDF原文件，某个同人专辑抓轨后得到的无损文件，某个东方Arrange的midi或mp3文件，某些同人文章的全文等。
    - 可以在词条中给出相关外部链接。


- 用户在其本人的“用户”和“用户Wiki”页面一般可以随意编写。

## 词条分类规范
- 词条分类必须准确。
- 请优先使用已有分类，请勿随意建立分类。已有分类请参阅分类列表 (未找到链接)。
- 如果使用游戏名、角色名作为分类的时候，必须使用全称。
- 关于如何给词条设置分类，请参阅[常用代码帮助](./帮助-常用代码.md)。

  
这些是 **错误** 的分类：
  

```
[[分类:风神录]]
[[分类:yyc]]
[[分类:灵梦什么的最讨厌了]]
```

  
这些是 **正确** 的分类：
  

```
[[分类:东方风神录]]
[[分类:博丽灵梦]]
```

## 角色词条内容规范
- 官方角色请按照固定规范格式进行，请参考[博丽灵梦](./博丽灵梦.md)词条。
- 为了表示清晰，请在词条头部以表格 (未找到链接)形式制作信息简介。
- 底部导航模块的制作请参见[高级代码帮助](./帮助-特殊代码.md)。
- 请遵守客观公正的原则，编写词条时候尽量避免无根据的个人看法。二次设定与一设在编写的时候需要区分开，可能引起争议或需要引用根据的内容请使用[出处标记](./帮助-特殊代码.md)。
- 为了提升角色词条阅读友好度，自2019年11月起，官方角色词条在角色信息模块下增设 **角色介绍** 部分，包括 **生活状况** 、 **角色能力** 、 **外貌特征** 、 **人际关系** 与 **登场剧情** 五个方面，对官方内容以及爱好者群体中与该角色有关的信息进行提炼，并用可读性更好的形式组织成文。其规范参见[帮助:角色介绍可读化](./帮助-角色介绍可读化.md)。
- 一设资料按照求闻系列设定集、官作游戏、其他游戏、文花帖（书籍）与口授的新闻、Fanbook、其他出版物、其他资料的顺序排列展示（其他资料单独开一个标题），同一类型则按时间顺序排列展示。
  - 对于原出处使用翻译表的资料，应使用ttt函数 (未找到链接)引用。

- 角色词条的词条导航必须包含全部该角色出场过的作品。

## 使用模板的说明
- 需要使用各种不确定翻译、或需要统一的用词时，请使用模板。
- 使用模板的情况通常有角色名称翻译、作品名称翻译、展会名称、音乐名称（以及翻译）和部分角色称号（以及翻译）。
- 标准名称模板共有分类：[标准名称模板](./分类-标准名称模板.md)
- 角色名称翻译和作品名称翻译模板均为直接使用的，不需要任何参数。
- 展会名称、音乐名称和部分角色称号则会需要附上参数。其中音乐名模板比较复杂，各参数对应值可以参考[音乐名模板目录](./音乐名模板目录.md)。
